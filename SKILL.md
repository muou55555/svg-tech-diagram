# SVG 技术示意图绘制与渲染

产出两个文件：可编辑的 SVG 源文件 + 高清 PNG（嵌入文档用）。核心思路：**SVG 是 XML 文本，结构化示意图用代码写比用绘图工具更精确、可复现、可迭代**。

## 工作流程

### 第 1 步：环境准备（每个新环境只需一次）

运行 `scripts/setup_env.sh`。它是幂等的：检查 cairosvg 与中文字体，缺什么装什么。沙箱环境通常没有 CJK 字体，直接渲染中文会变成方框（tofu），脚本通过 npm 的 @fontsource 包侧载 Noto Sans SC 解决（apt 通常无权限，这是少数可靠路径）。

```bash
bash <skill目录>/scripts/setup_env.sh
```

脚本结束会输出可用的中文字体族名（通常是 `Noto Sans SC Thin`），SVG 里的 `font-family` 必须与之一致。

### 第 2 步：设计布局（先想清楚再写）

在写 SVG 前先确定：画布尺寸（常用 1000×700 上下）、网格结构（如 2 行 × 3 列卡片）、各元素坐标。推荐纸面规划方式：容器留边距 40px，卡片间距 30–40px，行高统一。布局模式与配色方案见 `references/svg-patterns.md`——画之前读一遍，里面有容器+卡片网格、纵向流程、分层架构三种常用骨架的完整坐标模板。

### 第 3 步：手写 SVG

关键规则（违反会导致渲染失败或效果差）：

- **字体**：`<style>` 中声明 `text { font-family: 'Noto Sans SC Thin','Noto Sans SC',sans-serif; }`，族名以 setup 脚本输出为准
- **文字定宽估算**：中文字符宽度 ≈ font-size × 字数（全角），英文 ≈ font-size × 0.55 × 字符数。写每行文字前心算一下是否超出所在盒子，超了就换行（多个 `<text>` 错开 y）或缩小字号——cairosvg 不会自动换行
- **居中**：`text-anchor="middle"` + x 设为盒子中心
- **箭头**：用 `<marker>` 定义箭头头部，线条引用 `marker-end="url(#arrow)"`
- **不可用**：`foreignObject`（cairosvg 不支持）、emoji（无字体）、外部图片引用；渐变、圆角、虚线、透明度均可用
- 数值坐标保持整数网格对齐，同类元素尺寸严格一致——观感整齐主要靠这个

### 第 4 步：渲染 PNG

```bash
python3 <skill目录>/scripts/render_svg.py 输入.svg 输出.png --width 2000
```

`--width` 设为文档显示宽度的 2 倍（默认 2000），保证高分屏清晰。

### 第 5 步：视觉校验（不可省略）

**用 Read 工具查看生成的 PNG**，逐项检查：① 中文是否有方框/缺字 ② 文字是否溢出盒子或互相重叠 ③ 连线箭头指向是否正确 ④ 对齐与间距是否整齐。发现问题改 SVG 重渲染，通常 1–2 轮即可。这一步是质量保障的核心——SVG 是盲写的，只有渲染后才能确认效果。

### 第 6 步：交付

SVG 源文件与 PNG 一起保存（建议放目标目录的 `assets/` 子目录，文件名用英文避免引用问题）。若插入 Markdown，用相对路径 `![描述](./assets/xxx.png)`，并注明 SVG 源文件位置便于后续修改。

## 常见失败模式

| 症状 | 原因 | 处理 |
|---|---|---|
| 中文全是方框 | 字体没装好或 font-family 不匹配 | 重跑 setup 脚本，核对族名 |
| 文字超出盒子 | 没做定宽估算 | 按第 3 步公式重算，换行或缩字号 |
| 渲染报错 XML | 标签未闭合、& 未转义（写成 &amp;） | 检查 XML 语法 |
| 图发虚 | 输出宽度太小 | --width 至少为显示宽度 2 倍 |
| 布局歪斜 | 坐标随手写 | 回到第 2 步，用网格化坐标 |
